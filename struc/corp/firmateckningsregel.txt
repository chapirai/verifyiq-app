Mapping:

* typ -> type
* typ.kod -> type.code
* typ.klartext -> type.text
* ordning -> order
* funktionarsroller -> roles
* funktionarsroller[].kod -> roles[].code
* funktionarsroller[].klartext -> roles[].text
* funktionarer -> officers
* funktionarer[].personnamn -> officers[].personName
* funktionarer[].organisationsnamn -> officers[].organizationName
* funktionarer[].identitet -> officers[].identity

Rules:

* ordning: defines execution order, must be preserved exactly
* funktionarsroller: array of KodKlartext, do not flatten into strings
* funktionarer: array of nested officer objects, preserve structure
* this object exists inside a nested array structure (array[array]) and must not be flattened
* empty nested objects indicate separate schema definitions
* missing optional arrays should be [] if present but empty, otherwise null/omitted
