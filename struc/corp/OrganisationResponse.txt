* many sections are optional and only appear when requested or available
* empty objects in examples mean nested schema must be resolved separately before strict validation
* organisationsstatusar, funktionarer, samtligaOrganisationsnamn, aktieslag, organisationsengagemang, organisationsmarkeringar, godkandAvInstanser, frivilligaBestammelser, vakansOchUpplysning, finansiellaRapporter, rapporter are repeatable collections

* stammotidManader may contain null values and must allow null entries
* tillstand is documented as object, but permit information may need internal normalization to array if multiple permits are possible in practice
* numeric values for capital and counts should preserve precision and safe integer handling
* missing optional objects should be stored as null or omitted, not as empty placeholders
* code/text objects should be kept intact unless internal model splits them into code and display text fields
