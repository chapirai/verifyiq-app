* instance: string

* timestamp: string<date-time>

Mapping:

* type -> type
* instance -> instance
* status -> status
* timestamp -> timestamp
* requestId -> requestId
* title -> title
* detail -> detail
* code -> code
* source -> source

Rules:

* type conversion: convert timestamp to DateTime only if internal model requires it; keep status as integer
* optional/missing handling: type, instance, status, title required; detail, code, source, requestId, timestamp may be missing depending on error type
* special parsing: validate type against known values (about:blank, urn:bolagsverket:error:auth, urn:bolagsverket:error:network, urn:bolagsverket:error:validation)
